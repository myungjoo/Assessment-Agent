👤 사용자 입장

Web UI(React) + REST API 로 직접 사용 가능한 기능
  • 로그인 / 최초 SuperAdmin 셋업 최초 가입자는 SuperAdmin 자동 지정, 이후 가입은 User. 로그인 시 JWT 쿠키 발급
  • 사용자 관리 · 권한 승급/강등 Admin 이 사용자 추가, SuperAdmin 만 User↔Admin 변경(본인 self-demote 금지)
  • 평가 대상 인원 / 그룹 / 파트 CRUD 서비스별 ID 매핑, deactivate(휴직 숨김), 그룹 다중소속 · 파트 1개 소속
  • 평가 결과 대시보드 조회 (정렬·필터·페이지네이션) 이름·ID·지표별 정렬/필터, 점수 분포 히스토그램, 일/주/월 시계열 차트
  • LLM provider 설정 + 난이도 3슬롯 모델 매핑 Admin 이 5개 provider 등록(API key 암호화 저장), easy/medium/hard 슬롯 지정
  • 평가 진행 중 경고 배너 (R-78) 수집/평가 중에는 기존 자료만 표시 + 상단 경고. 단 자동 polling 은 미배선
  • 사용자 지정 기간 평가문 요청 (R-9) POST /assessment-evaluation/period 동작(User 휘발 / Admin 영속). UI 패널은 일부 defer
  • 재평가 / 스케줄 패널 backend 계약 일부 미shipped 로 ReEvaluation·Schedule 패널은 stub 상태
  • Import / Export / Restore 백업 Admin 패널 UI 는 존재하나 backend export/import 계약 진행 중(P7)

🧪 테스터 입장

로컬/CI 에서 검증 가능한 범위 (Node 20 LTS + pnpm 9)
  • 로컬 전체 검증 스위트 실행 아래 명령으로 lint → build → unit → smoke → e2e 까지 순차 검증
  • Unit 테스트 161개 + 커버리지 게이트 line ≥ 80% · function ≥ 80% · branch/statement ≥ 90% 미달 시 CI fail
  • Smoke 테스트 14개 NestJS 부트스트랩 + persons/groups/auth/llm/web-static 등 핵심 healthcheck
  • E2E 테스트 16개 status·header·body contract 검증 (auth/assessments/period-bridge/instance-access 등)
  • 실제 PostgreSQL 통합 검증 CI 가 postgres:16 컨테이너 + prisma migrate deploy 로 mock 아닌 real DB 검증
  • REST API 직접 호출 (~55 엔드포인트) 아래 엔드포인트 인벤토리 참고 — RBAC(User/Admin/SuperAdmin) 가드 포함
  • 토큰 암호화 CLI 시험 scripts/encrypt-token.ts 로 평문 토큰 → AES-256-GCM ciphertext 변환
  • 실 네트워크 LLM round-trip env-gated live spec (credential 부재 시 skip). Q-0022/live-LLM 검증 6/30 만료 전 필요
  • 부하/성능 테스트 (100~200명, 1시간 이내, R-91/92) P8 hardening 단계에서 진행 예정

⚡ 빠른 시작 (테스터)

pnpm install # 의존성 설치 pnpm lint # ESLint 정적 분석 pnpm build # NestJS 빌드 (TS → dist/) pnpm test # unit (Jest) pnpm test:cov # unit + coverage 게이트 pnpm test:smoke # smoke (부트스트랩 + 핵심 endpoint) pnpm test:e2e # e2e (HTTP contract) pnpm --filter web build # 프론트엔드 빌드 pnpm --filter web test # 프론트엔드 unit (vitest)
프론트엔드 개발 서버: cd web && pnpm dev (Vite, /api 는 localhost:3000 NestJS 로 proxy). 전체 규칙은 README 로컬 빌드/테스트 참고.
